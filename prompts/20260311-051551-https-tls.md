## Key decisions

**Certificate generation**: Used the `cryptography` library (already a transitive dependency) to generate RSA 2048-bit keys and self-signed X.509 certificates with SANs for localhost, *.localhost, *.amazonaws.com, and *.localhost.localstack.cloud. Certificates valid for 365 days.

**Cert caching**: Generated certs are written to `ROBOTOCORE_STATE_DIR/certs/` and reused on subsequent boots. This avoids slow key generation on every restart.

**HTTPS server**: Runs as a daemon thread via `uvicorn.Server` alongside the main HTTP server. Both serve the same ASGI app. The HTTPS thread is daemonic so it dies when the main process exits.

**Module-level TLS state**: `_tls_config` and `_tls_cert_path` are module-level variables in `app.py`, set by `main.py` during startup. This lets the `/_robotocore/tls/info` endpoint report cert details without coupling the endpoint to the startup flow.

**Test structure**: 18 unit tests for cert generation/config/caching, 5 integration tests for SSL context creation and the management endpoint. All tests are fast (no network, no server).
