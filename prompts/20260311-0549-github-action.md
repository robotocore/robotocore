## GitHub Actions composite action for setup-robotocore

Implemented a reusable GitHub Actions composite action that lets CI pipelines start a Robotocore container with one step.

### What was built

1. **Tests first** (`tests/unit/ci/test_setup_action.py`): 44 tests covering YAML structure, inputs/outputs, docker steps, env var mapping, cleanup script, and example workflow.

2. **Composite action** (`.github/actions/setup-robotocore/action.yml`): Pulls the GHCR image, starts container with configurable env vars (SERVICES, PERSISTENCE, ENFORCE_IAM, plus arbitrary KEY=VALUE configuration), waits for health check, and sets AWS env vars.

3. **Cleanup script** (`.github/actions/setup-robotocore/cleanup.sh`): Stops and removes the container.

4. **Example workflow** (`.github/workflows/example-robotocore.yml`): Reference workflow (workflow_dispatch only) showing basic and advanced usage with S3/SQS smoke tests.

5. **README** (`.github/actions/setup-robotocore/README.md`): Usage examples, input/output tables, environment variable documentation.

### Design decisions

- Used `GITHUB_ENV` for AWS credential injection so all subsequent steps inherit them automatically.
- Health check polls `/_robotocore/health` with 1-second intervals up to the configurable timeout.
- On timeout, dumps the last 50 lines of container logs for debugging.
- Configuration input supports arbitrary newline-separated KEY=VALUE pairs converted to `-e` flags.
