**This prompt log covers the `robotocore` CLI** (PR #108).

The CLI wraps Docker commands (`docker run/stop/logs`) and robotocore API calls
(`/_robotocore/state/*`, `/_robotocore/health`) behind a unified `robotocore`
command. Chose argparse + stdlib only (subprocess, urllib, json) to avoid adding
dependencies. The container name defaults to `robotocore-main` for predictable
lifecycle management.

`robotocore wait` polls the health endpoint every 1s — useful for CI where the
container needs to be healthy before tests run.
