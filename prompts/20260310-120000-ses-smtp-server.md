## Key decisions

**aiosmtpd over custom asyncio**: Used `aiosmtpd` library which provides a well-tested SMTP server implementation with a simple handler interface. The `Controller` class runs the server in a daemon thread, which integrates cleanly with the existing Starlette startup/shutdown lifecycle.

**Sender validation strategy**: The SMTP handler checks senders against Moto's SES backend verified identities (both email addresses and domains). When no identities are configured (common in dev), all senders are allowed for convenience. This matches real SES behavior where unverified senders are rejected.

**Email store as singleton**: Thread-safe `EmailStore` with a lock, since the SMTP server runs in a separate thread from the main ASGI server. Messages are stored with parsed fields (sender, recipients, subject, body) plus the raw email for debugging.

**Management endpoints**: Added `GET /_robotocore/ses/messages` and `DELETE /_robotocore/ses/messages` following the existing `/_robotocore/` prefix convention used by other management endpoints (chaos, resources, audit).

**Port 1025 default**: Standard non-privileged SMTP port, configurable via `SMTP_PORT` env var. Can be disabled entirely with `SMTP_DISABLED=1`.

**Start in `_start_background_engines`**: The SMTP server starts synchronously via `Controller.start()` (which spawns its own thread), alongside the existing Lambda event source engine and CloudWatch alarm scheduler. No need for async startup.
