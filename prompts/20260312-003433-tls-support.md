## Key decisions

**This prompt log covers HTTPS/TLS support** (PR #113).

**openssl subprocess over cryptography lib**: Generating self-signed certs via
`openssl req -x509 ...` avoids adding the `cryptography` package (which has
C extensions and platform-specific build requirements). The `openssl` binary is
available in the Docker container and on virtually all dev machines.

**Dual-port model**: HTTP on 4566 (always) + HTTPS on 4567 (when enabled). Chose
separate ports over upgrading the existing port because: (1) existing clients
don't break, (2) health checks can still use plain HTTP, (3) matches how real
AWS has separate HTTP/HTTPS endpoints.

**Three config levels**: `ROBOTOCORE_TLS=1` (auto self-signed, zero config),
`ROBOTOCORE_TLS_CERT/KEY` (explicit paths), or the existing `gateway.tls.TLSConfig`
(backward compat). The new env vars take priority, falling back to legacy config.

**Warning on self-signed**: Auto-generated certs print a clear warning that they're
for development only. The cert uses CN=localhost with a 365-day expiry.
