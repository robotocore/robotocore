## Key decisions

**Architecture**: Extracted shared logic into `cdklocal_lib.py` (pure functions,
no side effects) so both the bash and Python wrappers use the same logic, and
tests can import it directly without subprocess overhead.

**Two pure functions**: `build_env()` returns the environment dict, `build_command()`
returns the command list. Both are deterministic given inputs, making them trivial
to test.

**Bootstrap detection**: Simple `"bootstrap" in args` check. CDK subcommands are
always positional, so this is reliable.

**Credential handling**: Uses `dict.setdefault()` to never override user-set values.
This matches the principle of least surprise -- if a user has `AWS_ACCESS_KEY_ID`
set, we respect it.

**30 tests written first**: 23 unit tests covering each configuration axis
independently, plus 7 semantic/integration tests validating end-to-end behavior
and the "cdk not found" error path.
