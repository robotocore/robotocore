## Key decisions

**Test strategy**: Test each action function directly by calling it with a
real `SnsStore` instance and mock `Request` object. This avoids the async
`handle_sns_request` wrapper and tests the core logic synchronously.

**Cross-service delivery mocking**: Used `@patch` on the imported references
rather than the module-level names, since `_deliver_to_lambda` and
`_deliver_to_http` use lazy imports inside the function body:
- Lambda: patched `robotocore.services.lambda_.invoke.invoke_lambda_async`
- HTTP: patched `urllib.request.urlopen`
- SQS: patched `robotocore.services.sns.provider.get_sqs_store`

**ARN constants**: Extracted common ARN strings into module-level constants
(`SQS_ARN`, `TOPIC_NOPE`, `LAMBDA_ARN`, etc.) to keep lines under 100 chars
and reduce duplication across 113 tests.
