## Assistant

Created the DNS subsystem in `src/robotocore/dns/` with three components:

1. **resolver.py** -- Pure logic module that decides whether a hostname should resolve locally or be forwarded upstream. Supports `DNS_NAME_PATTERNS_TO_RESOLVE_UPSTREAM` (bypass patterns) and `DNS_LOCAL_NAME_PATTERNS` (extra local patterns). Handles A, AAAA (IPv4-mapped IPv6), and CNAME record types.

2. **server.py** -- UDP DNS server using dnslib. Runs in a background daemon thread. Forwards non-matching queries to upstream DNS (auto-detected from /etc/resolv.conf or configurable via `DNS_SERVER`). Handles malformed packets gracefully (returns SERVFAIL). Graceful fallback when port binding fails (e.g. port 53 needs root).

3. **Integration** -- Wired into `main.py` startup and `app.py` shutdown. Added `GET /_robotocore/dns/config` management endpoint. Added `dnslib>=0.9` to pyproject.toml.

Key design decisions:
- IPv4-mapped IPv6 addresses use pure hex notation (not mixed `::ffff:x.x.x.x`) for dnslib compatibility
- DNS server failure is non-fatal -- logs a warning and continues (port 53 requires root)
- Upstream bypass patterns take priority over local patterns
