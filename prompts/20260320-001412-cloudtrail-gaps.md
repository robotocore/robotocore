# CloudTrail Gap Implementation

## Task
Implement 10 missing CloudTrail operations in the Moto fork and write compat tests.

## Operations Implemented
1. **LookupEvents** - Returns empty events list (moto doesn't track real events)
2. **ListPublicKeys** - Returns empty public key list stub
3. **StartDashboardRefresh** - Validates dashboard exists, returns refresh ID
4. **StartEventDataStoreIngestion** - Sets EDS status to ENABLED
5. **StopEventDataStoreIngestion** - Sets EDS status to STOPPED_INGESTION
6. **GenerateQuery** - Returns stub query statement
7. **SearchSampleQueries** - Returns empty search results (SearchPhrase param, not SearchText)
8. **ListImportFailures** - Validates import exists, returns empty failures
9. **ListInsightsData** - Returns empty Events list (InsightSource/DataType params)
10. **ListInsightsMetricData** - Returns empty Timestamps/Values lists

## Key Decisions
- All new operations return stub/empty responses since moto doesn't simulate real CloudTrail event recording
- Checked botocore service model for correct parameter names (SearchPhrase not SearchText, InsightSource not EventSource for ListInsightsData)
- Checked botocore output shapes for correct response keys (ListInsightsData returns "Events" not "InsightsData")
- Dashboard GetDashboard output shape doesn't include Name field, only CreateDashboard does
