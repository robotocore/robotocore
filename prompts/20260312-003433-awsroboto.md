## Key decisions

**This prompt log covers `awsroboto`** (PR #109) — the `awslocal` equivalent.

**`os.execvp` over `subprocess.run`**: The wrapper replaces its own process with
`aws`, so signals (Ctrl-C), TTY behavior, and exit codes pass through correctly.
`subprocess.run` would require explicit signal forwarding and exit code propagation.

**Env var strategy**: Sets `AWS_ENDPOINT_URL` (not `--endpoint-url` arg injection)
because the env var works with all SDK versions and doesn't interfere with argument
parsing. Only sets credentials/region if not already set — respects user's existing
AWS config.

**No dependency on robotocore**: The wrapper is a standalone script that only needs
`aws` on PATH. You can install it via `pip install robotocore` and use it even if
the server is running elsewhere.
